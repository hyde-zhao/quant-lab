## 变更描述

实验十四 `reports/experiment_14/data_and_benchmark_report.md` 与实验十五因子框架验证已确认当前研究数据存在以下关键边界：

| 检查项 | 当前结论 | 风险 |
|---|---|---|
| 数据覆盖 | 本地 `data/prices.parquet` 覆盖 `2020-01-02` 至 `2024-04-24`，但阶段二 / 实验十三报告出现 `2025-01-02` 至 `2025-05-07` | 历史报告与当前标准 parquet 数据版本不一致，不能直接横向比较 |
| benchmark | 当前默认只能声明代理基准，非真实沪深300指数 | `benchmark_total_return` / `excess_return` 容易被误读为真实指数超额收益 |
| 股票池 | 当前 `index_members` 为固定快照池，`is_pit_universe=false` | 存在幸存者偏差，不适合直接支撑严肃因子结论 |
| 未来收益窗口 | 默认 `forward_return_horizon=20` 时，当前可用标签截止 `2024-03-25` | 覆盖终点附近样本不能生成完整未来收益标签 |
| 复权口径 | 当前 `prices.adjustment_policy=qfq` | 后续必须作为硬门禁，禁止混合复权口径静默进入实验 |
| 因子工程输入 | 实验十五已能使用 `close` / `volume` / 固定股票池生成因子面板和最小策略回测 | 只能证明框架可运行；缺少 PIT、可交易性、行业 / 市值、真实执行价与公司行动数据时，不能支撑严肃因子归因或可交易结论 |

本 CR 目标是把数据层从“可跑实验”升级为“可支撑研究结论”的最小闭环：统一研究数据入口、拆清代理 benchmark 与真实 benchmark、补 PIT 股票池路径、把复权 / 质量 / 未来收益窗口变成实验准入门禁，并沉淀统一的 research dataset builder。

本 CR 不替代 CR-007。CR-007 主要补齐 canonical 数据覆盖与真实 benchmark 数据生产；CR-008 在 CR-007 基础上强化研究消费侧合同、报告字段语义和准入门禁。

## 最终方案建议（2026-05-21）

### 总结结论

CR-008 的推荐方向是：继续以本项目 `market_data/` canonical 数据可信层为主干，不把 QUANTAXIS、Qlib、Backtrader、vn.py、NautilusTrader 或 LEAN 任一框架提升为核心数据层。外部项目只作为设计参考、数据源连接器或消费端适配目标。

最终形态应为：

```text
国内数据源连接器
Tushare / AKShare / TDX / 东方财富 / CSV / 未来付费源
        ↓
raw + manifest
保留源字段、源接口、请求参数、抓取时间、checksum、run_id
        ↓
canonical
统一 schema、复权、交易日历、benchmark、股票池、基础信息、质量状态
        ↓
research_input_v1
统一研究入口：价格、日历、股票池、benchmark、可交易性、metadata、gate result
        ↓
适配出口
自研 engine / Backtrader clean feed / Qlib exporter / VectorBT panel
```

本方案的核心判断：

| 判断项 | 最终建议 | 原因 |
|---|---|---|
| 核心数据层 | 保留并强化 `market_data/` canonical lake | CR-007/CR-008 的关键问题是 coverage、lineage、quality、PIT、benchmark 语义，不是回测框架替换问题 |
| 国内数据获取 | 以 Tushare 为结构化主源，AKShare / TDX / 东方财富作为补充源和交叉验证源 | Tushare API 形态更适合 manifest 与字段归一；AKShare 覆盖广但接口稳定性需质量门禁兜底 |
| QUANTAXIS | 借鉴 QAFetch / QAData 的国内多源与数据对象划分，不直接引入为核心依赖 | 其数据结构和 MongoDB / QAData 体系会与当前 `market_data` 分层重叠，仍需二次转换 |
| Backtrader | 仅作为 clean feed 消费端 | Backtrader Pandas feed 适合消费 OHLCV，不负责数据质量、复权、PIT 或 lineage |
| Qlib | 作为未来因子 / ML 研究导出目标，不作为主存储 | Qlib `.bin` 和 DataHandler 很适合研究，但不适合作为多数据源审计、raw/manifest 和多后端适配的唯一事实源 |
| 自研 engine | 继续作为默认日频研究主路径 | 当前项目目标是本地 A 股日频研究可信链路，而非完整实盘或多资产交易平台 |

### 数据源分工

| 数据类型 | 主源 | 辅源 | CR-008 使用方式 |
|---|---|---|---|
| 日线 OHLCV、复权因子 | Tushare | AKShare / TDX | 进入 `prices`、`adj_factor` canonical；复权口径必须单一且可追溯 |
| 交易日历 | Tushare `trade_cal` | AKShare | 作为 benchmark coverage 唯一分母；不得用自然日分母声明 benchmark pass |
| 沪深300指数 | Tushare `index_daily` | AKShare / 东方财富 | 真实 benchmark 只在同区间 coverage 合格时输出 `hs300_*` |
| 沪深300成分 / 权重 | Tushare `index_member` / `index_weight` | AKShare | `index_members` 与 `index_weights` 语义分离；权重不得静默替代完整成分 |
| 股票基础信息 | Tushare `stock_basic` | AKShare / 东方财富 | 支撑上市 / 退市 / ST / 市场状态门禁；缺 PIT 字段时不得伪装 PIT |
| 行业、市值、财务 | Tushare / AKShare | 后续付费源 | 作为 CR008-S06 辅助数据合同；缺失时阻断中性化、归因或容量结论 |
| Tick / 分钟 / 盘口 | 暂不作为 P0 | TDX / 付费源 | 当前项目主线是日频研究；高频能力后续单独 CR 评估 |

| Dataset | 用途 | 缺失时处理 |
|---|---|---|
| `trade_status` / `limit_status` / `st_status` | 停牌、涨跌停、ST、是否可买卖 | 只允许披露“未启用交易门禁”，不得声明真实可成交 |

### research_input_v1

CR-008 应新增统一研究入口 `research_input_v1`，建议落点为 `engine/research_dataset.py`。该入口只消费 `market_data.readers`，不得导入 connector/runtime/storage，不触发 backfill。

推荐请求合同：

```python
ResearchDatasetRequest(
    lake_root=...,
    start_date=...,
    end_date=...,
    universe="csi300",
    universe_mode="pit|required" 或 "fixed_snapshot|exploratory",
    benchmark_policy="hs300_required" 或 "proxy_allowed",
    adjustment_policy="qfq",
    forward_return_horizon=20,
    require_tradability=True/False,
    require_industry=False,
    require_market_cap=False,
)
```

推荐输出合同：

```python
ResearchDataset(
    prices=...,
    ohlcv=...,
    calendar=...,
    universe=...,
    benchmark=...,
    tradability=...,
    auxiliary=...,
    metadata=...,
    gate_result=...,
)
```

强制 metadata 字段：

| 字段 | 说明 |
|---|---|
| `manifest_run_id` / `source_run_id` | 数据 lineage，可为空但为空时必须触发 `lineage_unavailable` |
| `coverage_start` / `coverage_end` | 实际数据覆盖区间 |
| `label_available_end` | 考虑 `forward_return_horizon` 后的可用标签终点 |
| `adjustment_policy` | 单一复权口径，默认 `qfq`；混用必须 fail |
| `benchmark_kind` | `proxy` / `hs300` / `missing` / `policy_unconfirmed` |
| `universe_mode` | `pit` / `fixed_snapshot` / `missing` |
| `is_pit_universe` | 只有 PIT 字段完整且 as-of gate 通过时为 true |
| `quality_status` / `readiness_status` | 质量与可用性分离；quality pass 不等于 PIT available |
| `known_limitations` | 所有降级和不可支持结论必须结构化写入 |

### 准入门禁

| Gate | 失败行为 |
|---|---|
| 复权口径不唯一 | `fail`，禁止实验继续 |
| `forward_return_horizon` 超出可用未来收益 | `fail`，或显式截断样本并写入 `label_available_end` |
| 真实 benchmark 缺失 | 只能输出 `proxy_*` 字段，不得输出真实 `hs300_*` 超额收益 |
| PIT 股票池缺失 | 严肃因子研究 `fail`；探索模式可降级为 `fixed_snapshot`，但必须写幸存者偏差说明 |
| 交易日历缺失 | benchmark coverage fail；不得用自然日分母冒充通过 |
| 可交易性缺失 | 报告只能声明“未启用交易门禁”，不得声明真实可成交 |
| 行业 / 市值 / 风格暴露缺失 | 不得声明行业中性、市值中性、风格中性或纯 alpha 结论 |

建议统一状态枚举：

```text
quality_status: pass | warn | fail | missing
dataset_status: available | unavailable | required_missing | quality_failed
readiness_status: available | warn | unavailable | pit_incomplete | non_pit_snapshot
pit_status: pit_available | pit_incomplete | non_pit_snapshot | pit_failed
benchmark_status: real_available | proxy_only | missing | policy_unconfirmed
```
